# Marathon Bot — Telegram-бот для навчального марафону

Бот для керування 20-денним платним марафоном у Telegram: стежить за чатом, нагадує
про невідповідені повідомлення учасників, аналізує настрій через Claude API, пропонує
готові відповіді, публікує відкладені пости й веде рейтинг активності з балами.

## Як влаштований проєкт

```
marathon_bot/
├── main.py                  — точка входу, запуск бота
├── config.py                 — читання налаштувань з .env
├── db/database.py            — SQLite: учасники, повідомлення, бали, черга постів
├── services/claude_service.py — виклики Claude API (тональність, детекція завдань, відповіді)
├── bot/handlers.py            — обробка повідомлень групи + callback-кнопки
├── bot/admin_commands.py       — /start, /leaderboard, /bonus, /schedule
├── bot/notifier.py             — формування сповіщень адміну
├── scheduler/jobs.py           — нагадування, публікація постів, щоденний дайджест
└── tests/test_scoring.py       — юніт-тести логіки балів (без Telegram/мережі)
```

## Функціонал

1. **Моніторинг чату** — бот читає всі повідомлення учасників (текст і фото з підписом).
2. **Нагадування** — якщо повідомлення адресоване організаторці і вона не відповіла
   протягом `REMINDER_THRESHOLD_HOURS` годин (за замовчуванням 6) — нагадування в приватний
   чат з 2 варіантами відповіді. Термінові повідомлення (фрустрація/ризик покинути марафон)
   сповіщаються одразу.
3. **Відкладені пости** — команда `/schedule YYYY-MM-DD HH:MM [текст]` (можна у відповідь
   на пост із фото).
4. **Аналіз настрою + щоденний дайджест** — категорії: positive, neutral, frustrated,
   question, urgent. Дайджест надсилається щодня о `DAILY_DIGEST_TIME`.
5. **Пропозиції відповідей** — 2 варіанти теплим, підтримуючим тоном, з кнопками
   "Надіслати варіант 1/2" — надсилає бот від свого імені прямо в групу.
6. **Бали й `/leaderboard`**:
   - +1 за повідомлення в основному чаті
   - +1 за коментар у гілці обговорення (`message_thread_id`)
   - +10 за підтверджене виконання завдання дня — **детектується за хештегом `#деньN`**
     (напр. `#день5`), захищено від подвійного зарахування (унікальний запис на
     учасника+день у базі)
   - `/bonus <бали> [причина]` у відповідь на повідомлення учасника, або
     `/bonus <user_id> <бали> [причина]`

Усі команди керування (`/start`, `/leaderboard`, `/bonus`, `/schedule`) працюють лише
для твого `ADMIN_USER_ID` — інші учасники їх просто не побачать.

## Крок 1. Створення бота через @BotFather

1. Напиши [@BotFather](https://t.me/BotFather) в Telegram → `/newbot`.
2. Вкажи ім'я і username бота (має закінчуватись на `bot`).
3. Отримаєш **токен** виду `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` — це `BOT_TOKEN`.

### ⚠️ Обов'язково вимкни Privacy Mode

За замовчуванням бот у групах бачить **лише** повідомлення, адресовані йому напряму
(команди, reply, згадки). Щоб бот бачив усі повідомлення учасників (це потрібно для
моніторингу, нагадувань і балів) — вимкни Privacy Mode:

1. У @BotFather: `/mybots` → вибери свого бота → **Bot Settings** → **Group Privacy** → **Turn off**.
2. Якщо бот вже доданий у групу — вийди і додай його знову (або хоча б призначь
   адміністратором групи), щоб зміна набула чинності.

## Крок 2. Дізнатись ID групи і свій user_id

Найпростіше — через [@userinfobot](https://t.me/userinfobot) або [@RawDataBot](https://t.me/RawDataBot):

- **Свій user_id (`ADMIN_USER_ID`)**: напиши `@userinfobot` в особисті — він покаже твій `id`.
- **ID групи (`GROUP_CHAT_IDS`)**: додай `@RawDataBot` у групу марафону, він одразу пришле
  JSON з `"chat": {"id": -100123456789, ...}`. Це значення (разом зі знаком `-`) і є ID групи.
  Після перевірки бота можна видалити з групи.
  - Якщо у тебе канал з окремою групою обговорень (коментарі до постів каналу) — тобі
    потрібен ID саме **групи обговорень** (там з'являються коментарі учасників), а не каналу.
  - Можна вказати кілька ID через кому: `GROUP_CHAT_IDS=-100111,-100222`.

## Крок 3. Ключ Anthropic API

1. Зареєструйся на [console.anthropic.com](https://console.anthropic.com/).
2. **Settings → API Keys → Create Key**.
3. Поповни баланс (Billing) — без цього запити повертатимуть помилку.
4. Скопіюй ключ виду `sk-ant-...` — це `ANTHROPIC_API_KEY`.

## Крок 4. Налаштування проєкту

```bash
cd marathon_bot
cp .env.example .env
```

Відкрий `.env` і заповни:

```
BOT_TOKEN=<токен від BotFather>
ANTHROPIC_API_KEY=<ключ Anthropic>
ADMIN_USER_ID=<твій user_id>
GROUP_CHAT_IDS=<ID групи (груп) через кому>
REMINDER_THRESHOLD_HOURS=6
TIMEZONE=Europe/London
DAILY_DIGEST_TIME=20:00
```

**Ніколи не комітьте `.env` у git** — він уже доданий у `.gitignore`.

## Крок 5. Встановлення і локальний запуск

Потрібен Python 3.11+.

```bash
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

Якщо все налаштовано правильно, в консолі з'явиться `Бот запускається...`, і бот
почне отримувати повідомлення. Напиши щось у групу і перевір:

- чи бот бачить повідомлення (`REMINDER_THRESHOLD_HOURS` спрацює нескоро — можна
  тимчасово поставити `0.01` для тесту);
- напиши `/leaderboard` у приватний чат з ботом (від імені `ADMIN_USER_ID`) — має
  показати рейтинг;
- напиши в групі щось з `#день1` — має нарахуватись +10 балів.

## Крок 6. Запуск юніт-тестів

Логіка нарахування балів протестована без реального Telegram/Claude API:

```bash
pip install -r requirements-dev.txt
pytest tests/ -v
```

Усі 13 тестів мають пройти (перевірка нарахування балів, захисту від подвійного
зарахування завдань, детекції хештегів `#деньN`, рейтингу, логіки нагадувань).

## Крок 7. Розгортання на Railway.app (24/7)

1. Заведи акаунт на [railway.app](https://railway.app/) (можна через GitHub).
2. У Railway: **New Project → Deploy from GitHub repo** → вибери цей репозиторій.
3. У **Variables** додай усі змінні з твого `.env` (`BOT_TOKEN`, `ANTHROPIC_API_KEY`,
   `ADMIN_USER_ID`, `GROUP_CHAT_IDS`, `REMINDER_THRESHOLD_HOURS`, `TIMEZONE`,
   `DAILY_DIGEST_TIME`, `CLAUDE_MODEL`, `DB_PATH`, `TASK_HASHTAG_DAYS`) — **не** через
   файл `.env` (він не потрапить у репозиторій), а напряму в інтерфейсі Railway.
4. У **Settings → Start Command** вкажи:
   ```
   python main.py
   ```
5. Railway автоматично визначить Python-проєкт за `requirements.txt` і встановить
   залежності. Після деплою бот працюватиме 24/7.

⚠️ **Про базу даних**: за замовчуванням бот зберігає дані в SQLite-файлі (`marathon.db`)
на диску контейнера. На безкоштовному плані Railway диск не гарантовано зберігається
між передеплоями — щоб дані (бали, історія) не губились, підключи **Railway Volume**
(Settings → Volumes → додай volume, змонтуй, наприклад, у `/data`, і встанови
`DB_PATH=/data/marathon.db` у змінних середовища).

## Обмеження поточної реалізації

- **Детекція завдань дня** працює лише за хештегом `#деньN` (1-20), як домовились —
  без хештегу повідомлення не зарахується як звіт про виконання, навіть якщо це
  очевидний скріншот результату. Якщо захочеш пізніше додати розпізнавання без
  хештегу через Claude API — це окрема доробка.
- **"Адресовано мені"** визначається комбінацією reply/тегу учасника на повідомлення
  організаторки та класифікацією Claude API — це евристика, можливі поодинокі
  хибні спрацювання чи пропуски.
- **+1 бал за коментар у гілці обговорення** визначається по наявності
  `message_thread_id` у повідомленні Telegram (тред/гілка). Якщо структура групи інша
  (наприклад, звичайна група без топіків і без зв'язаного каналу), усі повідомлення
  зараховуватимуться як "+1 за повідомлення в чаті".
- **Один адміністратор** — команди керування прив'язані до одного `ADMIN_USER_ID`,
  як домовились. Додати кількох модераторів можна пізніше, змінивши перевірку в
  `bot/admin_commands.py` (`admin_only`) на список ID.
- Кнопки "Надіслати варіант 1/2" публікують відповідь **від імені бота**, а не від
  твого особистого акаунту — учасники бачитимуть, що відповів бот-асистент, а не ти
  особисто.
- Немає веб-інтерфейсу чи адмінки — усе керування через Telegram-команди.
- SQLite підходить для одного марафону з одним ботом-інстансом; для паралельного
  запуску кількох інстансів (масштабування) знадобиться перехід на PostgreSQL.
